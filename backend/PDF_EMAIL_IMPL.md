## Approve Flow (POST /api/requisitions/:id/approve)

```
1. findById()           → validate exists + status === 'drafted'
2. updateStatus()       → status: 'approved', approvedAt, approvedBy
3. auditService.log()   → event: 'requisition_approved'
4. findByIdRich()       → joins: rhu → municipality, approvedBy → users, breachId → threshold_breaches
5. generateRequisitionPdf()  → PDFKit → PassThrough stream → Buffer (in-memory, no disk writes)
6. sendRequisitionEmail()    → Resend SDK → HTML body + PDF attachment
7. updateStatus()       → status: 'sent', sentAt: now
8. auditService.log()   → event: 'requisition_sent'
9. return updated row
```

**Error handling:** PDF or email failures are non-fatal — approval is committed before PDF/email.
If either fails, the error is logged and the `approved` status is returned.

## `findByIdRich()` — Joins

```sql
SELECT
  requisitions.id,
  rhu.name            AS rhuName,
  rhu.barangay,
  municipalities.name AS municipalityName,
  municipalities.province,
  users.name          AS mhoName,          -- LEFT JOIN on approvedBy
  requisitions.approved_at,
  threshold_breaches.projected_zero_date   -- LEFT JOIN on breach_id
FROM requisitions
INNER JOIN rhu             ON rhu.id            = requisitions.rhu_id
INNER JOIN municipalities  ON municipalities.id  = rhu.municipality_id
LEFT JOIN  users           ON users.id           = requisitions.approved_by
LEFT JOIN  threshold_breaches ON threshold_breaches.id = requisitions.breach_id
WHERE requisitions.id = $1
```

Then a second query fetches items (genericName, unit, quantityRequested, currentStock).
